# Content Structure

## Start with Purpose, Not Structure

- Don't start from the header and work down
- Ask: what is the key functionality or main selling point of this website?
- It could be a heading, input field, button, or image
- Design as little as possible first — start minimal, add only what's needed

## Page Section Anatomy

A typical web page flows through these sections:

### 1. Navigation Header

- Logo (top-left), primary links, search, CTA/login (top-right)
- Keep it simple: 5-7 main links max
- Sticky on scroll for easy access

### 2. Hero / Above the Fold

- Most visitors never scroll past the fold — make it count
- Include: clear headline, supporting text, primary CTA
- Crystal clear value proposition
- After designing, scroll backward (bottom to top) for a fresh perspective

### 3. Social Proof / Trust

- Logos of clients/partners, testimonials, stats
- Placed early to build credibility
- 2-3 strong testimonials > 10 generic ones
- Show human faces and real names

### 4. Features / Benefits

- What does this product/service do for the user?
- Use 2-3 column layouts with icons + short descriptions
- Focus on benefits (what users GET), not features (what it DOES)

### 5. Detailed Content / How It Works

- Deeper dive for interested users
- Process steps, comparisons, detailed explanations
- Use progressive disclosure: summary first, details on expand

### 6. Social Proof Continued / Case Studies

- More detailed testimonials or success stories
- Data and results

### 7. Pricing / CTA

- Clear pricing or next steps
- Remove decision fatigue: highlight the recommended option
- Final strong CTA

### 8. FAQ

- Address common objections
- Reduces support load

### 9. Footer

- Secondary navigation, legal links, contact info
- Social media links
- Don't overlook it — `<nav>` and `<footer>` tags matter for SEO

## HTML Document Structure for SEO

- One `<h1>` per page — summary of the entire page
- `<h2>` tags divide major sections
- `<p>` for body text
- `<ul>`/`<ol>` and `<li>` for lists
- Google reads structure before content
- Proper hierarchy helps search engines AND screen readers

## Content Writing for the Web

- Action-oriented button text: "Download the guide" not "Submit"
- Tell people exactly what happens when they click
- "See pricing" / "Start free trial" / "Download the guide" > "Learn more" / "Click here"
- Use headings that communicate value, not just labels
- Write scannable text: short paragraphs, bold keywords, clear subheadings

## URL and Link Management

- Descriptive, readable URLs: `/pricing` not `/page?id=42`
- Set up 301 redirects when changing URLs
- Fix broken links promptly — they hurt SEO and trust
- Descriptive alt text on every image (not just for accessibility, also SEO)

## Information Density

- More information ≠ better. Give users what they need to make a decision
- Competing priorities: you can't show everything at once
- Budget: decide what matters most for each section/page
- Use progressive disclosure for deep content: summary visible, details on click/expand

## Navigation Architecture

- Keep primary navigation to 5-7 items
- Use clear, descriptive labels
- Mega menus for complex sites, simple dropdowns for smaller ones
- Always include search for content-heavy sites
- Breadcrumbs for deep hierarchies
- User flow > information architecture: optimize for the shortest path to the goal
